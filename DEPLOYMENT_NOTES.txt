계정 보안 설정·관리자/사용자 세션 정책 배포 안내

기준 패키지
- system_management_json_restore_deployment_package.zip

배포 패키지 구조
- src/
- rules/firestore.rules
- tools/
- DEPLOYMENT_NOTES.txt
- VALIDATION_REPORT.txt

중요 경로 변경
- Firestore Rules 전체 교체본은 프로젝트 루트가 아니라 rules/firestore.rules에 저장합니다.
- 이후 Rules 관련 배포본도 동일한 rules 폴더 구조를 사용합니다.

배포 순서
1. 배포 패키지의 src 폴더를 프로젝트 src 폴더에 반영합니다.
2. 배포 패키지의 rules/firestore.rules를 로컬·원격 프로젝트의 rules/firestore.rules 전체 교체본으로 반영합니다.
3. 현재 프로젝트에서 사용하는 Firebase 배포 설정이 rules/firestore.rules를 참조하는지 확인합니다.
4. Firestore Rules를 배포합니다.
5. 최고 관리자 계정으로 관리자 모드에 로그인합니다.
6. 시스템 > 계정 보안 설정에서 관리자 세션과 사용자 세션의 초기값을 확인하고 저장합니다.
7. 기존 로그인 세션이 정책 버전 변경으로 다시 로그인되는지 확인합니다.
8. 사용자 로그인과 관리자 로그인을 각각 시험하여 브라우저 지속성과 자동 로그아웃을 확인합니다.

메뉴 변경
- 시스템 관리 내부의 '관리자 보안' 탭을 제거했습니다.
- 시스템 메뉴 아래에 독립 메뉴인 '계정 보안 설정'을 추가했습니다.

  시스템
  ├─ 시스템 관리
  └─ 계정 보안 설정

- 시스템 관리는 사이트 설정, 서비스 운영, 데이터 점검·백업·복원, 초기화, 시스템 정보, 변경 이력을 담당합니다.
- 계정 보안 설정은 관리자와 일반 사용자의 로그인 지속성 및 세션 만료 정책을 담당합니다.

계정 보안 설정 권한
- 최고 관리자(owner): 조회·변경 가능
- 일반 관리자(admin): 조회만 가능
- 정책 저장 시 systemAuditLogs에 변경 이력을 기록합니다.

관리자 세션 기본값
- 브라우저·탭 종료 시 로그아웃: 사용
- 무활동 자동 로그아웃: 60분
- 1회 로그인 최대 유지시간: 8시간

관리자 세션 입력 범위
- 무활동 자동 로그아웃: 15~480분
- 1회 로그인 최대 유지시간: 0~168시간

사용자 세션 기본값
- 브라우저·탭 종료 시 로그아웃: 사용 안 함
- 무활동 자동 로그아웃: 120분
- 1회 로그인 최대 유지시간: 24시간

사용자 세션 입력 범위
- 무활동 자동 로그아웃: 15~1440분
- 1회 로그인 최대 유지시간: 0~168시간

최대 유지시간 0의 의미
- 0은 애플리케이션의 절대 최대 세션 제한을 사용하지 않는다는 뜻입니다.
- 무활동 자동 로그아웃과 브라우저·탭 종료 시 로그아웃은 각각 설정된 상태로 계속 적용됩니다.
- Firebase ID 토큰의 자체 만료나 갱신 구조를 변경하는 값은 아닙니다.

브라우저·탭 종료 시 로그아웃
- 사용: Firebase browserSessionPersistence와 sessionStorage를 사용합니다.
- 사용 안 함: Firebase browserLocalPersistence와 localStorage를 사용합니다.
- 관리자와 사용자 로그인 직전에 해당 역할의 정책으로 Firebase 인증 지속성을 다시 설정합니다.
- 동일 브라우저에서 관리자와 일반 사용자의 동시 로그인을 지원하지 않으며, 다른 역할 로그인 시 기존 역할의 애플리케이션 세션을 정리합니다.

무활동 자동 로그아웃
- 클릭, 키보드, 스크롤, 터치, 탭 복귀를 활동으로 감지합니다.
- 잦은 저장을 막기 위해 세션 갱신은 최소 30초 간격으로 제한합니다.
- 활동 시 무활동 만료시각만 연장하고 최초 로그인 기준의 절대 만료시각은 연장하지 않습니다.

정책 버전
- 관리자 세션 설정이 바뀌면 adminSecurityPolicyVersion이 증가합니다.
- 사용자 세션 설정이 바뀌면 userSecurityPolicyVersion이 증가합니다.
- 현재 세션에 저장된 버전과 최신 정책 버전이 다르면 Firebase 로그아웃 후 재로그인을 요구합니다.

Firestore 문서
- 관리자 세션 정책: systemSettings/admin
- 사용자 세션 정책: securityPolicies/userSession
- securityPolicies/userSession은 로그인 전에도 읽어야 하므로 공개 읽기를 허용합니다.
- 두 문서의 세션 정책 변경은 최고 관리자만 가능합니다.

JSON 백업·복원
- JSON 백업에 systemSettings/admin과 securityPolicies/userSession을 포함합니다.
- 복원 화면의 '계정 보안 정책' 영역은 최고 관리자만 선택할 수 있습니다.
- 관리자 ID, 권한, 비밀번호, 현재 로그인 세션은 복원하지 않습니다.
- 세션 정책 복원 시 백업의 정책 버전을 그대로 덮어쓰지 않고 현재 버전보다 증가시켜 기존 세션을 무효화합니다.
- 계정 보안 정책은 데이터 복원과 건전성 검사가 끝난 마지막 단계에서 원자적으로 반영합니다. 복원 도중 현재 최고 관리자 세션이 먼저 종료되는 문제를 방지하기 위한 처리입니다.

로그인 공격 보호 안내
- Firebase Authentication의 기본 남용 방지와 요청 제한을 사용합니다.
- 클라이언트가 자체적으로 실패 횟수를 기록하는 '5회 실패 후 계정 잠금'은 구현하지 않습니다.
- 로그인 오류 통합 표시와 비밀번호 재설정 계정 존재 은폐는 앱에 적용되어 있습니다.
- Firebase 서버 비밀번호 정책과 공식 이메일 열거 방지의 실제 활성 상태는 Firebase Console에서 확인해야 합니다.

운영 확인 시나리오
1. 관리자 종료 시 로그아웃 ON 상태에서 탭·창 종료 후 관리자 재접속
2. 관리자 종료 시 로그아웃 OFF 상태에서 브라우저 재실행 후 세션 복구
3. 관리자 무활동 만료
4. 관리자 최대 유지시간 만료
5. 관리자 최대 유지시간 0에서 활동 중 절대 만료 없음
6. 사용자 종료 시 로그아웃 ON/OFF 각각의 브라우저 재접속
7. 사용자 무활동 만료
8. 사용자 최대 유지시간 만료 및 0 설정
9. 관리자 정책 변경 후 기존 관리자 재로그인 요구
10. 사용자 정책 변경 후 기존 사용자 재로그인 요구
11. 일반 관리자에서 설정 입력 비활성화 확인
12. JSON 백업에 계정 보안 정책 포함 확인
13. JSON 복원 후 관리자·사용자 정책 버전 증가 확인

배포 검증 제한
- 기준 패키지에는 package.json과 node_modules가 없어 실제 npm run build를 실행하지 못했습니다.
- 현재 작업 환경에는 Firebase CLI와 Emulator가 없어 Rules 공식 컴파일 및 실제 프로젝트 인증 지속성 시험을 수행하지 못했습니다.
- 운영 반영 전 테스트 프로젝트 또는 별도 테스트 계정에서 위 시나리오를 확인하십시오.
