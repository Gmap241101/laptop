사이트 기본 설정·시스템 메뉴 분리 배포 안내

기준 패키지
- account_security_session_policy_deployment_package.zip

이번 변경 범위
1. 사이트 관리 첫 메뉴로 '사이트 기본 설정'을 독립 배치했습니다.
2. 사이트 기본 설정에 있던 기본 메인 비주얼 편집 기능을 '홈 화면 기본 설정'으로 분리했습니다.
3. 기존 메인 비주얼 관리, 프로모션 배너 관리, 바로가기 배너 관리는 통합하지 않고 그대로 유지했습니다.
4. 기존 '시스템 관리' 메뉴를 제거하고 다음 독립 메뉴로 분해했습니다.
   - 서비스 운영
   - 계정 보안 설정
   - 데이터 관리
   - 시스템 정보·로그
5. 데이터 관리는 내부 탭으로 '점검·백업·복원'과 '데이터 초기화'를 제공합니다.
6. 시스템 정보·로그는 내부 탭으로 '시스템 정보'와 '변경 이력'을 제공합니다.
7. 다른 대여 운영, 커뮤니티, 사용자·권한 메뉴는 이번 배포에서 통폐합하거나 명칭을 변경하지 않았습니다.

최종 관리자 메뉴 중 변경 영역

사이트 관리
- 사이트 기본 설정
- 홈 화면 기본 설정
- 메인 비주얼 관리
- 프로모션 배너 관리
- 바로가기 배너 관리
- 팝업 관리
- 푸터 관리

시스템
- 서비스 운영
- 계정 보안 설정
- 데이터 관리
- 시스템 정보·로그

사이트 기본 설정
- 사이트 정식 명칭·짧은 명칭·기관명
- 브라우저 제목과 사이트 설명
- 헤더 부제·대표 주소
- 로고·모바일 로고·파비콘
- 기본·진한 브랜드 색상
- 서비스 문의 정보
- 헤더와 브랜드 요소 실시간 미리보기

홈 화면 기본 설정
- 등록된 메인 비주얼이 없을 때 사용할 기본 메인 비주얼 사용 여부
- 기본 제목
- 기본 설명
- 홈 화면 기본 콘텐츠 미리보기

설정 저장 분리
- 사이트 기본 설정 저장 시 사이트 기본정보 필드만 갱신합니다.
- 홈 화면 기본 설정 저장 시 기본 메인 비주얼 필드만 갱신합니다.
- 서비스 운영 저장 시 운영모드·기능별 접수·전역 안내 필드만 갱신합니다.
- 각 영역 저장은 별도 감사 로그 항목으로 기록됩니다.
- 다른 영역의 동시 설정값을 전체 문서로 다시 덮어쓰지 않도록 부분 필드 저장을 적용했습니다.

읽기 최적화
- 사이트 기본 설정, 홈 화면 기본 설정, 서비스 운영 화면에서는 시스템 변경 이력과 초기화·복원 작업 문서를 구독하지 않습니다.
- 데이터 관리 화면에서만 초기화·복원 작업 상태를 구독합니다.
- 시스템 정보·로그 화면에서만 최근 변경 이력 50건을 구독합니다.

기존 메뉴 키 호환
- 이전 'settings' 메뉴 키로 진입하면 '서비스 운영' 메뉴로 자동 전환합니다.

미저장 변경 보호
- 사이트 기본 설정, 홈 화면 기본 설정, 서비스 운영에서 저장하지 않고 다른 메뉴로 이동하면 확인창을 표시합니다.

배포 순서
1. 배포 패키지의 src 폴더를 프로젝트 src 폴더에 반영합니다.
2. 배포 패키지의 rules/firestore.rules를 현재 rules/firestore.rules와 비교합니다.
3. 이번 변경에서는 Firestore 경로와 권한을 변경하지 않았으므로 Rules 내용은 기준본과 동일합니다.
4. 프로젝트를 빌드합니다.
5. 관리자 화면에서 사이트 관리와 시스템 메뉴의 순서와 화면을 확인합니다.
6. 사이트 기본 설정, 홈 화면 기본 설정, 서비스 운영을 각각 수정·저장하여 다른 영역 값이 유지되는지 확인합니다.
7. 데이터 관리의 두 내부 탭과 시스템 정보·로그의 두 내부 탭을 확인합니다.

주의
- Firestore 데이터 마이그레이션은 필요하지 않습니다.
- siteSettings/config 문서 구조는 변경하지 않았습니다.
- rules 파일은 앞으로 rules/firestore.rules 경로로 제공합니다.
