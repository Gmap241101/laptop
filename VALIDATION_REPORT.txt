- 최고 관리자 전용 타 프로젝트 강제 복원
- 프로젝트 ID 및 스키마 버전 검사
- 회원 개인정보 미포함 백업의 회원영역 복원 차단
- 현재 데이터와 문서 ID 충돌 및 백업 외 현재 문서 수 미리보기
- 자산 참조 오류와 중복 자산관리번호 사전 차단
- 복원 직전 개인정보 포함 전체 자동 백업
- 점검 모드 자동 전환 및 복원 완료 후 점검 모드 유지

- systemRestoreJobs 기반 중단 작업 재개
- 복원 후 데이터 건전성 검사와 감사 로그
- 관리자 ID, 관리자 Auth, 초기화·복원 작업·감사 로그 복원 제외

검증 제한
- 기준 패키지에는 package.json과 node_modules가 없어 실제 npm run build를 실행하지 못했습니다.
- 현재 환경에는 Firebase CLI/Emulator가 없어 Firestore Rules 공식 컴파일과 실제 Firebase 프로젝트 쓰기 테스트를 수행하지 못했습니다.
- 웹 클라이언트는 다른 사용자의 Firebase Authentication 계정 존재 여부를 조회할 수 없습니다. 회원 Firestore 문서 복원 후 Auth 계정 유무는 Firebase Console 또는 별도 Auth 백업으로 확인해야 합니다.

운영 배포 전 필수 시험
1. 동일 프로젝트 백업의 없는 문서만 추가
2. 같은 ID 병합 및 덮어쓰기
3. 최고 관리자 선택 영역 초기화 후 복원
4. 일반 관리자에게 초기화 후 복원 방식 비활성화 확인
5. 프로젝트 ID 불일치 시 일반 관리자 차단
6. 최고 관리자 강제 복원과 대상 프로젝트 ID 확인
7. 복원 중 네트워크 중단 후 같은 파일로 재개
8. 개인정보 제외 백업에서 회원영역 비활성화
9. 자산 참조 오류가 있는 백업의 실행 차단
10. 복원 완료 후 점검 모드 및 데이터 건전성 검사
